Developer Guide

Architecture, implementation, and extension workflow

This page documents the engineering side of the project: local setup, source layout, API behavior, webhook scripting, security controls, and CI/CD.

1) Development Setup

Area Required Notes
Client (Tauri) Rust stable, cargo tauri Linux needs WebView/GTK packages and xfreerdp3 or xfreerdp for RDP.
Server Python 3.12+, pip FastAPI + SQLAlchemy + SQLite.

Install Tauri CLI

4) Client Architecture (Tauri)

Module File Responsibility
Command bridge commands.rs Exposes load/save/open/sync/password commands to frontend.
Connection openers connection/* SSH/RDP/Web launch behavior per OS.
Local storage storage.rs Stores connections.json and settings.json in app data folder.
Sync sync.rs Validates and fetches sync source; HTTPS only.
Password handling password.rs RDP only; keyring/Credential Manager integration.
Important: local CRUD is blocked in Sync mode by the frontend.

5) Server Architecture (FastAPI)

Entry point: server/app/main.py. At startup, DB tables are created and an initial admin user is created if the DB is empty.

Area Files Details
Auth auth.py, routers/auth.py JWT bearer auth, bcrypt with SHA-256 pre-hash.
DB models database.py, models.py Users, API keys, webhook metadata in SQLite.
Connections storage.py, routers/connections.py Connections are file-based JSON, not SQL rows.
IP filtering middleware.py IP/CIDR allowlist with optional trusted proxy support.
Webhooks routers/webhooks.py, script_runner.py Python script execution; full builtins including import are available (admin-only feature).

6) Extension Architecture

  • background.js: refresh alarm every 5 minutes, fetch + cache + badge update.
  • popup.js: setup flow, search, list/tag views, open tab actions.
  • options.js: settings + connection test.
  • chrome.storage.sync for config and chrome.storage.local for cached connection data.

7) API and Authentication Behavior

Two auth modes exist: JWT bearer for users and API key via X-API-Key header or api_key query parameter.

Endpoint Access Notes
POST /api/auth/login public Returns access token.
GET /api/connections JWT or API key (read/read_write) Returns all connections.
POST/PUT /api/connections Admin JWT or API key read_write Create/update; UUID is generated on create.
DELETE /api/connections/{id} Admin JWT only Delete is stricter than write key mode.
/api/users, /api/api-keys, /api/webhooks Admin JWT only System administration domain.

8) Webhook Scripting

Trigger endpoint: POST /api/webhooks/trigger/<token>. Scripts run server-side with full Python builtins — including import — plus injected storage helpers and request context. Because webhooks are admin-only, no builtin restrictions apply.

Runtime object Type Use
load_connections() function Read all current connections from JSON storage.
save_connections(list) function Persist modified connection list.
uuid4() function Create a new UUID string.
payload, headers, params dict Request data for the current trigger call.
result, logs, log(msg) dict/list/function Build response payload and runtime logs.
import ... statement All Python modules available, e.g. import requests, import json, import re.

Example: create or update a connection

Repository file: docs/examples/webhook_add_connection.py 

connections = load_connections()
incoming = payload.get("connection")

if not isinstance(incoming, dict):
    raise ValueError("payload.connection must be an object")

# normalize + validate + upsert
save_connections(connections)
result["mode"] = mode
result["connectionId"] = connection_id

Example: import connections from an HTTP API

Repository file: docs/examples/webhook_http_import.py 

import requests

api_url = payload.get("api_url", "https://cmdb.example.com/api/servers")
r = requests.get(api_url, timeout=10)
r.raise_for_status()

connections = load_connections()
added = 0
for srv in r.json():
    if not any(c.get("host") == srv["ip"] for c in connections):
        connections.append({
            "id": uuid4(),
            "name": srv["hostname"],
            "host": srv["ip"],
            "kind": "ssh",
            "port": 22,
            "tags": srv.get("tags", []),
        })
        added += 1

save_connections(connections)
result["added"] = added
log(f"{added} connections imported from {api_url}")
Note: Scripts run synchronously in the request thread. Always set a timeout on HTTP calls to avoid blocking the webhook endpoint indefinitely.

  "trustCert": false,
  "lastUsed": "2026-01-27T10:39:21.574Z"
}

Client settings

  • mode: local | sync
  • url: sync source URL (HTTPS)
  • intervalMinutes: 1..1440
  • storePasswords: local RDP keychain usage
  • allowSelfSignedCerts: optional TLS cert bypass for sync requests

10) Security Controls in Code

  • Passwords are not stored inside connection JSON; RDP credentials are optional and local-only.
  • API keys are stored as SHA-256 hashes on server side.
  • IP filtering supports CIDR and trusted proxy chains.
  • Client sync accepts HTTPS URLs only.
  • Webhook scripts run with full Python builtins (including import) inside an isolated namespace with only injected storage functions exposed.
Recommendation: in proxy setups, set TRUSTED_PROXIES instead of enabling global TRUST_PROXY_HEADERS=true.

11) CI/CD Pipeline (.gitlab-ci.yml)

  • docker: builds and pushes server image to registry.
  • build: creates Linux binary and Debian package for client.
  • publish: optional Debian publishing via Aptly API script.

Publishing script: client/scripts/ci_publish_via_api.sh (requires mTLS and Aptly environment variables).

12) Updating

Update each component in a controlled, repeatable sequence. Server updates are image-based; client updates are build/distribution based.

Update server

docker compose pull
docker compose up -d

Create a backup of data/, certs/, and .env first.

Update client

cd client/src-tauri
cargo tauri build

Distribute the new build and recheck runtime dependencies (for example FreeRDP on Linux).

Recommendation: verify sync, auth, and webhook behavior after every version update.

13) Typical Extension Workflow

  1. Update JS model in client/src/connectionModel.js or settingsModel.js.
  2. Update Rust structs in client/src-tauri/src/models.rs.
  3. Expose command in commands.rs and register in main.rs.
  4. Adjust server schema/routes if API contract changes.
  5. Validate local mode and sync mode behavior end-to-end.

14) Troubleshooting

Sync fails

Client requires HTTPS. For self-signed certs, enable the matching client option.

RDP launch fails on Linux

Check xfreerdp3 / xfreerdp availability in PATH.

Server login issues

Initial ADMIN_PASSWORD only applies when DB is empty.

Unexpected 403

Review ALLOWED_IPS and TRUSTED_PROXIES.
