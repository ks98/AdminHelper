Developer Guide

API Reference

All HTTP endpoints of the AdminHelper server. Base URL: https://<host>. The complete OpenAPI schema lives at /api/docs (Swagger UI) and /openapi.json.

Authentication

Two mechanisms, depending on the caller:

  • JWT bearer (web panel, desktop): Authorization: Bearer <access_token>. Both access and refresh tokens are returned in the JSON body – no httpOnly cookie.
  • API key (agents, extension, third parties): header X-API-Key. Permission is read or read_write.

Auth

POST/api/auth/login

Body: { "username", "password" }. Response: { "access_token", "refresh_token", "user" }. Login rate limit: 5 failed attempts / 60 s / IP.

POST/api/auth/refresh

Body: { "refresh_token" }. Rotates both tokens, blacklists the old jti. Reusing an already-revoked refresh token invalidates all sessions of the affected user chain.

POST/api/auth/logout

Blacklists the current access token (and any refresh token in the body).

GET/api/auth/me

Current user; requires a valid access token.

POST/api/auth/bootstrap

Creates the first admin user via a setup token on first start (status 201). Body: { "token", "username", "password" } – username 3–64 chars, password 8–128 chars. Response: { "access_token", "refresh_token", "token_type" } (same as login). The setup token is written to the WARNING log on first start (hash stored at DATA_DIR/.bootstrap_token) and consumed after a successful bootstrap. The same login rate limit applies (5 failed attempts / 60 s / IP). Once any user exists, the endpoint permanently returns 409 Conflict ("Server already initialised"); if no active token is present it returns 401.

Users

GET/api/users

List all users (admin).

POST/api/users

Create user (admin). Password: SHA-256 prehash → bcrypt. Fires user.created.

PUT/api/users/{id}

Update user (admin).

DELETE/api/users/{id}

Delete user (admin). Fires user.deleted.

GET/api/users/me/servers

Servers assigned to the current user.

Connections

GET/api/connections

List (read).

POST/api/connections

Create (write). Fires connection.created.

PUT/api/connections/{id}

Update (write). Fires connection.updated.

POST/api/connections/{id}/touch

Sets last_used = now (read).

DELETE/api/connections/{id}

Delete (admin). Fires connection.deleted.

GET/api/connections/export

JSON export (admin).

POST/api/connections/import

?mode=replace|merge (admin). Fires connections.imported.

Servers

GET/api/servers

Server inventory (admin).

POST/api/servers

Create (admin). Fires server.created.

GET/api/servers/{id}
PUT/api/servers/{id}

Update (admin). Fires server.updated.

DELETE/api/servers/{id}

Delete (admin). Also triggers cleanup at the monitoring service.

API Keys

GET/api/api-keys
POST/api/api-keys

Body: { name, permission: "read"|"read_write" }. Clear-text key is returned once; only the SHA-256 hash is persisted.

DELETE/api/api-keys/{id}

Hooks

GET/api/hooks
POST/api/hooks

Body: { name, hook_type: "webhook"|"event"|"schedule", event_triggers?: [...], schedule_interval?, script, enabled }. For webhook the token is generated server-side (returned once).

GET/api/hooks/{id}
PUT/api/hooks/{id}
DELETE/api/hooks/{id}

Auto-removes from the scheduler if it was a schedule hook.

POST/api/hooks/{id}/toggle

Enable/disable; updates scheduler registration.

POST/api/hooks/{id}/rotate

Rotate webhook token.

POST/api/hooks/{id}/run

Manual test run with empty context.

POST/api/hooks/trigger/{token}

Public (no JWT). Rate limit: 20 req / 60 s / IP. Context: { payload, headers, params }.

GET/api/ansible/playbooks/{id}

FRP – Server Config

GET/api/frp/server-config
POST/api/frp/server-config

Also writes frps.toml into the shared volume.

GET/api/frp/server-config/{id}

Optional ?include_tunnels=true.

PUT/api/frp/server-config/{id}
DELETE/api/frp/server-config/{id}

FRP – Tunnels

GET/api/frp/tunnels

Query: ?server_id, ?frp_config_id.

POST/api/frp/tunnels

STCP (with secret_key, visitor_port) or HTTPS (custom_domains). auto_create_connection=true also creates a matching connection.

GET/api/frp/tunnels/{id}
PUT/api/frp/tunnels/{id}
DELETE/api/frp/tunnels/{id}

FRP – Generate (TOML/ZIP)

GET/api/frp/generate/frps-toml?config_id=…
GET/api/frp/generate/frpc-toml/{server_id}
GET/api/frp/generate/visitor-toml?config_id=…&user_id=…
GET/api/frp/generate/visitor-bundle?config_id=…

Returns { toml, pki }.

GET/api/frp/generate/bulk-zip?config_id=…

ZIP with frps.toml + clients/*.toml + visitors/*.toml.

Server provisioning (FRP-independent, since v0.23.2)

POST/api/servers/{server_id}/provision/token

Admin-only (JWT). Returns { token, expiresAt, serverId, serverName } – one-time, 24 h TTL.

GET/api/servers/{server_id}/provision/tokens
POST/api/servers/{server_id}/provision/activate

Header X-Provision-Token. Response: { serverName, apiKey, monitorApiKey?, monitorUrl?, frp? }. apiKey is always the one-time generated server read API key. monitorApiKey/monitorUrl are null when the monitor service is unreachable; frp is null when no FRP server config exists or the server has no tunnels. When set, frp has the shape { config, pkiBundle }: config is the generated frpc.toml as a Base64 string, pkiBundle a Base64-encoded tar.gz containing pki/ca.crt, pki/<servername>.crt and pki/<servername>.key (empty string "" when no PKI files are available).

FRP – sync (running operation)

GET/api/frp/provision/{server_id}/config

Current frpc.toml plain text (JWT or API key) – only for the running sync.

GET/api/frp/provision/{server_id}/config-hash

SHA-256 for drift-free synchronisation.

FRP – Status & PKI

GET/api/frp/status

Merges the frps dashboard with DB tunnels.

GET/api/frp/pki/status
POST/api/frp/pki/ca?common_name=…
POST/api/frp/pki/server-cert
POST/api/frp/pki/client-cert/{name}
GET/api/frp/pki/download/{filename}
GET/api/frp/pki/download-client-bundle/{client_name}

ZIP with ca.crt, client.crt, client.key.

Monitoring (reverse proxy)

POST/api/monitoring/agent/{server_id}/report

Public (X-API-Key). Forwards the agent push 1:1 to the monitoring microservice.

GET/api/monitoring/{path:path}

Admin-only reverse proxy. Allowed path prefixes: checks, alerts, log, metrics, status, templates, agent. Path traversal is blocked.

POST/api/monitoring/{path:path}
PUT/api/monitoring/{path:path}
DELETE/api/monitoring/{path:path}
No dedicated alerts, audit or settings endpoints on AdminHelper

Alerts and check status come through the monitoring proxy from the separate service. Audit trails run via hook events through the logging system; there is no dedicated /api/audit endpoint as of v0.23.2.

Error Format

json
HTTP/1.1 422 Unprocessable Entity
Content-Type: application/json

{
  "detail": [
    { "loc": ["body", "port"], "msg": "ensure this value is less than 65536", "type": "value_error" }
  ]
}

Errors follow Pydantic validation conventions.

Rate Limits

  • Login: 5 failed attempts / 60 s / IP (Redis with in-memory fallback).
  • Webhook trigger: 20 requests / 60 s / IP.
  • Agent push: not throttled, but every request must match its own server_id via X-API-Key.
Use Swagger directly

/api/docs renders OpenAPI 3.1 interactively, including "Try it out". For CI clients, a single openapi-generator generate -g go -i https://srv/openapi.json is enough.
