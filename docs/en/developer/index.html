Developer Guide

Overview

This guide is aimed at engineers working on AdminHelper itself. It walks through the four surfaces – server, web panel, desktop client, Go agent – plus the integration layers: monitoring, API, hooks, and CI/CD.

Tech Stack

Server

FastAPI 0.11, SQLAlchemy 2 + Alembic, APScheduler, Redis. 100+ pytest tests.

Frontend

Svelte 5 Runes, TypeScript strict, Vite 5, Playwright E2E.

Desktop

Tauri 2, Rust 1.83, Svelte 5, 41 Vitest unit tests.

Agent

Go 1.24, Cobra CLI, cross-compiled Linux + Windows DEB/RPM/MSI.

Local Development

bash
# Server
cd apps/server
uv sync
uv run uvicorn app.main:app --reload

# Web panel
cd apps/web
npm install
npm run dev

# Desktop
cd apps/desktop/ui && npm install
cd ../desktop  && cargo tauri dev

# Go agent
cd apps/agent
go run ./cmd/adminhelper-agent run

Unified PKI & mTLS

AdminHelper runs its own internal PKI as the foundation for mTLS across all entrances. The full, canonical model (decisions D1–D11, trust, lifecycle, backup) is in ADR 0001; the implementation status in ADR 0002. In brief:

  • Trust chain: one Root → scoped intermediates tunnel (frps + agents), access (humans: desktop/browser), internal (phase B). Clients pin the Root/CA, not a leaf — server certs rotate freely (D2).
  • ca-issuer (own container) is the only signing capability (D6): creates the hierarchy on first boot, holds the online intermediates, signs via /enroll (one-time token) and /renew (presented cert). The Root key stays cold + passphrase-encrypted (D7).
  • gateway (nginx) is the single public TLS edge (`:443` data, `:8444` enroll). It terminates TLS/mTLS and forwards the verified identity as X-Client-* headers to the internal services — uvicorn can't see the cert itself (V1). server/ca-issuer have no host port, so the header is unforgeable.
  • Enrollment (one pattern, three clients): one-time token → on-device ECDSA key + CSR → issuer signs (identity issuer-dictated, the CSR subject cosmetic). Go agent (at provisioning), desktop (after login), frps/tunnel certs. Native clients auto-renew at ~50% lifetime; the browser gets a long-lived cert as a PKCS12 export from the desktop (D5).
  • Per-route scope: access = human, tunnel = agent (D8). Permissive by default (`MTLS_ENFORCE=false`): a scope mismatch is logged, the request proceeds. Flipping to CERT_REQUIRED means MTLS_ENFORCE=true — implemented + end-to-end verified (A8), a deliberate operator decision.
  • Backup crown jewel is the ca-pki volume (Root + intermediates) — scripts/backup.sh/restore.sh back it up; the regenerable cert volumes are deliberately excluded (ADR 0001 §5).
