Web Panel

The Web Panel is a standalone Svelte 5 SPA with a hash router. It lives under frontend-src/, is built via Vite, and is multi-stage-copied into the Python runtime image at Docker build time.

Routing

Hash router (no SSR, no SvelteKit client router): paths are #/connections, #/servers, etc. This works reliably behind any reverse proxy without SPA fallback support on the server side. Implementation in src/lib/router.ts – a writable _path with a hashchange listener; exports navigate(), replace(), currentPath() and a segments derived store.

Eight production routes: #/connections, #/servers, #/users, #/apikeys, #/hooks, #/frp, #/ansible, #/monitoring.

Stores

Ten stores in src/lib/stores/. Each is a TypeScript class with $state fields and methods that talk to the matching API wrapper.

StorePurpose
authLogin, logout, refresh, current user; tokens in localStorage (adminhelper_token, adminhelper_refresh_token).
connectionsLive list, search, group filter, tag hierarchy.
serversServer inventory, provision token, user assignment.
usersUser CRUD incl. server mapping.
apikeysKey CRUD, one-time clear-text reveal.
hooksHook CRUD, trigger test, token rotation.
frpTunnel CRUD, provisioning, PKI status.
ansiblePlaybook management and editor state.
monitoringChecks, alerts, templates, VictoriaMetrics queries via proxy.
notificationsGlobal toasts and confirmation dialogs.

API Wrapper

All HTTP calls go through src/lib/api/client.ts. The client owns the access token, holds an in-flight-deduplicated tryRefresh() promise and automatically refreshes on 401. If that fails, registerAuthFailureHandler() calls into the auth store, which navigates to login.

Domain wrappers sit alongside: auth.ts, connections.ts, servers.ts, users.ts, apikeys.ts, hooks.ts, frp.ts, ansible.ts, monitoring.ts – all typed via types.ts.

i18n

Languages: DE and EN. Active language is persisted in localStorage.adminhelper_language; browser detection only supplies the initial fallback. The t() function is a store – this avoids i18n leaks in reactive components.

uPlot Charts

Monitoring charts use uplot directly – no heavyweight chart framework. Wrappers in src/lib/components/. uPlot is 30 KB gzipped and renders 100 k points without stutter.

Tests

Playwright smoke tests cover all 8 production pages. Two specs: login.spec.ts and smoke.spec.ts; visual diffs live in tests/e2e/*.spec.ts-snapshots/. Tolerance is maxDiffPixelRatio: 0.02 (2 %). Locale de-DE, viewport 1280×800. Reports are uploaded as a CI artifact (frontend_e2e job).

bash
cd frontend-src
npm run check           # svelte-check + tsc
npm run lint            # eslint + svelte-eslint
npx playwright test     # E2E

Build & Deployment

The Vite build outputs to frontend-src/dist/ (sourcemaps on, target ES2022). The repo-root Dockerfile uses a multi-stage build to copy the output into /app/frontend/, and the FastAPI app mounts it as a static route. SPA fallback checks file existence before returning index.html – this prevents MIME errors on /assets/*.js.

In dev mode Vite runs on port 5173 and proxies /auth, /api, /health to localhost:8000.
