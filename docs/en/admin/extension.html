Browser Extension

The AdminHelper extension adds a toolbar popup to Chromium-based browsers that launches HTTP/HTTPS connections directly. Manifest V3, two permissions, no content scripts.

Deployment

  1. Download the ZIP from Settings → Extensions or Web Panel /downloads/adminhelper-extension.zip.
  2. Unpack to a stable path (e.g. /opt/adminhelper-ext/).
  3. Chrome/Edge → chrome://extensions → Developer mode → Load unpacked.
  4. Open the extension options → enter Server URL and API key (scope web_only).

Rollout in Organizations

Managed Chrome policies can preinstall the extension:

json
{
  "ExtensionInstallForcelist": [
    "<ext-id>;https://srm.example.com/downloads/extension-update.xml"
  ]
}

The update XML lives next to the ZIP download and is served with the correct MIME type by the AdminHelper server.

Permissions

  • storage – Server URL and API key in chrome.storage.sync; cached connections in chrome.storage.local.
  • alarms – 5-minute refresh job in the background service worker.
Use a scoped API key

chrome.storage.sync is not a secret store. Always issue the extension a key with scope web_only – never give it connection-write or hook-admin privileges.

Protocol Filter

The popup shows only http and https connections. RDP/SSH/VNC entries are filtered out client-side – no URI-handler hijacking surface.
