Agent Deployment

One Go binary, two packages (DEB + RPM), shipped with matching service units. The provision flow redeems a one-time token and installs a persistent client certificate.

Debian / Ubuntu

bash
curl -fsSL https://deb.ks98.de/pubkey.asc | sudo tee /etc/apt/trusted.gpg.d/ks98.asc
echo 'deb https://deb.ks98.de/srm stable main' | sudo tee /etc/apt/sources.list.d/srm.list
sudo apt update
sudo apt install adminhelper-agent

RHEL / Rocky / Fedora

bash
sudo tee /etc/yum.repos.d/srm.repo <<'EOF'
[srm]
name=AdminHelper
baseurl=https://deb.ks98.de/srm-rpm/stable
gpgcheck=1
gpgkey=https://deb.ks98.de/pubkey.asc
EOF
sudo dnf install adminhelper-agent

Windows

Grab the latest MSI from the Servers page (Provision → Windows) or directly from https://srm.example.com/downloads/adminhelper-agent.msi. Install with elevated privileges – the installer registers the Windows service.

Provisioning

In the Web Panel: Servers → New → Provision Token. The token is valid for 24 h and redeemable exactly once.

bash
# Linux
sudo adminhelper-agent provision \
  --server https://srm.example.com \
  --token  TOKEN_VALUE
powershell
# Windows (PowerShell, as Admin)
& 'C:\Program Files\AdminHelper\adminhelper-agent.exe' provision `
    --server https://srm.example.com `
    --token  TOKEN_VALUE

After provisioning the agent holds a client certificate signed by the FRP CA and enables the adminhelper-agent.timer systemd unit (Linux) or the Windows service (auto-start).

Verification

bash
systemctl status adminhelper-agent.timer
journalctl -u adminhelper-agent.service -n 50
adminhelper-agent run --verbose
Migration from legacy agents

The DEB/RPM packages carry Replaces + Conflicts for srm-frpc-client, srm-monitor-agent, and srm-agent. A single apt install adminhelper-agent removes the old trio and preserves the config.
