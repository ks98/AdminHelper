Troubleshooting

Quick diagnostic paths for the five most common failure classes: login, TLS, agents, monitoring, and FRP.

Login fails

  1. Rate-limited? Check docker compose logs server | grep ratelimit – 5 bad attempts / 15 min blocks the IP.
  2. Password forgotten? Set ADMIN_PASSWORD=… in .env, restart, log in, clear it out again.
  3. Refresh cookie rejected? Check the clock skew between the browser and server (TOTP also relies on this).

TLS / reverse proxy

bash
# Cert valid?
openssl s_client -connect srm.example.com:443 -servername srm.example.com < /dev/null 2>/dev/null | openssl x509 -noout -dates

# X-Forwarded-For trusted?
docker compose logs server | grep 'untrusted proxy'

Agent doesn't check in

bash
adminhelper-agent run --verbose   # one-shot debug run
systemctl status adminhelper-agent.timer
journalctl -u adminhelper-agent.service -n 100 --no-pager

Common causes: clock skew > 5 min, expired client cert, DNS failure, firewall blocking 7443/tcp.

Monitoring ingest dropping samples

bash
# Check auth
docker compose logs monitoring | grep 'invalid api key'

# Check VictoriaMetrics health
docker compose exec monitoring curl -s http://victoria:8428/health

# Disk full?
docker compose exec monitoring df -h /data/victoria

FRP visitor connect hangs

  • Visitor-port pool exhausted? Scale VISITOR_PORT_END.
  • Client cert expired? Check adminhelper-agent frpc rotate.
  • Wrong sk on STCP tunnels – both sides must match.

Support Bundle

Collect a reproducible context for support requests:

bash
docker compose exec server python -m app.cli support-bundle \
  --output /tmp/adminhelper-bundle.tgz
Bundle may contain secrets

The support bundle includes .env-style config + the last 1000 log lines. Review before sharing – obfuscate API keys, passwords, and IPs that shouldn't leave the org.
