What is AdminHelper?

AdminHelper connects three worlds: a central web panel (FastAPI + Svelte 5) for managing all connections and monitoring data, a desktop client (Tauri 2 + Svelte 5) for opening sessions via RDP, SSH, VNC, and HTTP, and a Go agent that runs on Linux and Windows servers, reports metrics, and manages FRP tunnels.

Web Panel

Svelte 5 + strict TypeScript, hash router, i18n DE/EN, 8 production pages, uPlot charts, Playwright E2E tests.

Desktop Client

Tauri 2 + Rust, 24 typed commands, keyring secrets, RDP race guard, FRP sidecar, auto-update via DEB/RPM.

Go Agent

Unified Cobra CLI for Linux + Windows, collector framework for CPU/RAM/Disk/SMART/Docker, systemd timer with jitter.

Monitoring

APScheduler jobs push metrics to VictoriaMetrics; 10 check types including SMART health, HTTP, ping, SSH, Docker.

FRP Tunnel with mTLS

Own 10-year CA, 1-year client certificates, automatic provisioning via one-time token and systemd.

Auth & Security

JWT access with refresh rotation, Redis rate limit with fallback, reuse detection for stolen tokens.

What's new in 0.23.1?

v0.23.1 – Bundle release (2026-05-02)

Coordinated version bump across all components (server, web panel, desktop client, Go agent, browser extension, Docker image, CI pipeline) onto a shared version. No functional changes – just realigns every artifact after the big migrations from 0.17 and 0.19.

Highlights from 0.19.0

Desktop Client ported to Svelte 5

Big-bang migration in 11 phases: stores instead of DOM managers, strict TypeScript, typed Tauri bridge to all 23 Rust commands, 41 Vitest unit tests. The old desktop/src/ was replaced by desktop-src/. Details in the Developer Guide › Desktop Client.

Security hardening

Refresh-token rotation with blacklist and reuse detection, login rate limit on Redis (with in-memory fallback), Tauri capability scoping (minimal permissions), PKI bundle protected against zip-slip and zip-bomb. See Developer Guide › Server.

Monitoring dashboard v2

Section-based per-server dashboard instead of a card grid; heartbeat, live, network, services, Docker, backups, ZFS and SMART are grouped. Each check row expands into period tabs (1h·6h·24h·7d) with chart and status timeline.
