What is AdminHelper?

AdminHelper connects three worlds: a central web admin panel (FastAPI + Svelte 5) for instance management (users, API keys, hooks, FRP server configuration), a desktop client (Tauri 2 + Svelte 5) for opening sessions via RDP, SSH, VNC, and HTTP that, in server mode, is also the operational cockpit for server inventory, connections, tunnels, monitoring, and Ansible, and a Go agent that runs on Linux and Windows servers, reports metrics, and manages FRP tunnels.

Web Admin Panel

Svelte 5 + strict TypeScript, hash router, i18n DE/EN. Pure instance management: users, API keys, hooks, FRP server configuration. Playwright E2E tests.

Desktop Client

Tauri 2 + Rust, typed commands, keyring secrets, RDP race guard, FRP sidecar, auto-update via DEB/RPM. In server mode the operational cockpit: Infrastructure, connections, monitoring, Ansible.

Go Agent

Unified Cobra CLI for Linux + Windows, collector framework for CPU/RAM/Disk/SMART/Docker, systemd timer with jitter.

Monitoring

APScheduler jobs push metrics to VictoriaMetrics; 10 check types including SMART health, HTTP, ping, SSH, Docker.

FRP Tunnel with mTLS

Own 10-year CA, 1-year client certificates, automatic provisioning via one-time token and systemd.

Auth & Security

JWT access with refresh rotation, Redis rate limit with fallback, reuse detection for stolen tokens.

What's new in 0.33.0?

v0.33.0 – Desktop: password on every start, local mode & pre-fill (2026-06-14)

The desktop client now always asks for a password on open (no silently restored session) — with the server URL and username pre-filled. A new "continue without a server" button drops straight into local mode. Also fixed: local connections were overwritten by the server/sync fetch — local mode now has its own store.

v0.32.1 – logout no longer locks you out (2026-06-13)

The desktop client cleared the mTLS cert on logout — under enforced mTLS you then couldn't log back in ("certificate not sent"). The device cert now survives; only the session is dropped.

v0.32.0 – the operational cockpit moves into the desktop (2026-06-13)

Operational management has moved from the web into the desktop client: server inventory, connections, FRP tunnels, monitoring checks, and Ansible playbooks are now maintained in the server-centric "Infrastructure" hub (server mode only). The web admin panel stays focused on pure instance management: users (the default page after login), API keys, hooks, and the FRP server configuration. The user↔server assignment stays in the web panel (user dialog).

v0.31.0 – browser cert: pick the save location (2026-06-13)

Exporting the browser certificate (Desktop → Settings) now opens a save dialog — you choose where the .p12 is written instead of digging it out of a hidden app-data directory.

v0.30.4 – installer catches a stale Postgres volume (2026-06-13)

Re-installing over an old postgres-data volume (from an aborted attempt) failed with a cryptic 240s timeout, because Postgres only sets the password on first init. The installer now detects this and names the fix; a new --reset wipes old volumes up front (docker compose down -v).

v0.30.3 – reproducible image pinning (2026-06-13)

An install now pins the images to the installed version (--ref vX.Y.Z → :X.Y.Z) instead of tracking the floating :latest — reproducible, no silent version jump on docker compose pull. Upgrading is a deliberate ./scripts/update.sh --ref vNEW.

v0.30.2 – installer fix for curl | bash (2026-06-13)

The documented one-liner aborted instantly at the first prompt under curl | bash — read consumed the script pipe instead of your terminal. The interactive prompts (domain/password/confirm) now come from /dev/tty; without a terminal a clear message points to --admin-password … --yes.

v0.30.1 – update flow documented (2026-06-13)

The installation docs (DE+EN) now describe the update.sh flow with its two modes: refresh images only (./scripts/update.sh) versus a version upgrade with --ref (which also re-downloads the compose + scripts) — plus .env image pinning and the self-update caveat (update.sh does not overwrite itself).

v0.30.0 – one-command installer + self-contained compose (2026-06-12)

Install with one command (curl … install.sh | bash): the installer fetches only the runtime files (the docker-compose.yml is now self-contained — the stack creates its monitoring DB itself, no repo-file bind mounts), brings it up, creates the first admin + token, done. No source-tree clone needed. update.sh can refresh the runtime files per version.

v0.29.0 – enforced mTLS by default + one-shot installer (2026-06-12)

mTLS is now enforced by default (MTLS_ENFORCE=true), made safe by a new one-shot installer: scripts/install.sh brings up the stack, creates the first admin with a one-time enrollment token out-of-band (via a new in-container CLI) and then arms enforcement — so a fresh install is usable enforced right away, with no manual permissive window. Plus scripts/update.sh (backup-first → pull → healthcheck).

v0.28.0 – PKI/mTLS hardening + FRP visitor migration (2026-06-12)

A hardening pass over the internal PKI/mTLS surface: immediate identity revocation (renewal refusal and a per-request data-plane cut-off when a user/server is deleted), crash-safe cert renewal, automatic re-mint of the gateway/frps leaf before expiry, cleanup of spent enrollment tokens, and a rate limit on the certless enroll plane :8444. The desktop STCP visitor is migrated onto the unified PKI (it presents its enrolled access identity); the old server-run FRP CA and its PKI admin UI are gone – the server no longer signs certificates (D6).

v0.27.0 – Audit release: hardening + scale (2026-06-10)

Implements a full technical audit: two remaining TOML-injection gaps closed, migration smoke tests (the Alembic chain now runs in CI), new CI gates (Windows compile, tauri build smoke, docker PR builds, weekly dependency audit), ruff for Python. Scale preparation for 250–500 servers: API pagination, FK indexes, throttled agent service inventory, scheduler tuning, alert-log retention. The web monitoring page auto-refreshes; agent packages install to /usr/bin. Over 130 new tests.

v0.26.0 – FRP 0.69.1 (2026-06-07)

Bumps FRP from 0.61.1 to 0.69.1 (frps image and bundled frpc in lockstep, SHA-256 pinned). The wire protocol stays v1, so it is backward compatible. Tunnel-tested (STCP + visitor + mutual TLS).

v0.25.0 – Security hardening release (2026-06-06)

Closes 7 security advisories and 13 functional bugs: desktop zip-slip + TLS enforcement, CA-key isolation from the internet-facing frps, tunnel-free monitoring with no host port, JWT revocation on password reset, non-root containers + CI supply-chain pinning, agent TOFU instead of a persisted --insecure. Breaking (ops): the monitoring port 8480 is gone (re-provision agents); containers now run as non-root.

v0.24.0 – Security hardening & move to GitHub (2026-06-04)

A security release plus a move of the project infrastructure to GitHub: hardened FRP PKI key permissions (0600), closed IDOR and TOCTOU gaps in provisioning, an honest hook posture, and periodic JWT-blacklist cleanup. CI/CD now runs on GitHub Actions and the Docker images are served from the GitHub Container Registry (ghcr.io/ks98/adminhelper/…).

Highlights from 0.19.0

Desktop Client ported to Svelte 5

Big-bang migration in 11 phases: stores instead of DOM managers, strict TypeScript, typed Tauri bridge to all 23 Rust commands, 41 Vitest unit tests. The old desktop/src/ was replaced by apps/desktop/ui/. Details in the Developer Guide › Desktop Client.

Security hardening

Refresh-token rotation with blacklist and reuse detection, login rate limit on Redis (with in-memory fallback), Tauri capability scoping (minimal permissions), PKI bundle protected against zip-slip and zip-bomb. See Developer Guide › Server.

Monitoring dashboard v2

Section-based per-server dashboard instead of a card grid; heartbeat, live, network, services, Docker, backups, ZFS and SMART are grouped. Each check row expands into period tabs (1h·6h·24h·7d) with chart and status timeline.
