Agent-Deployment

Der Go-Agent läuft auf jedem überwachten Server. Er übernimmt zwei Aufgaben: Metrik-Collector-Läufe und FRP-Client-Auto-Provisioning. Ausgeliefert wird er als DEB, RPM (Linux) oder MSI/ZIP (Windows).

Was macht der Agent?

  • Monitoring: läuft alle 5 Minuten (systemd.timer mit Jitter), sammelt CPU, RAM, Disk, SMART, Docker-Container-Status, ZFS-Pools, sendet an /api/monitoring/ingest.
  • FRP-Client: lädt beim Erst-Start mit einem Provision-Token sein Client-Zertifikat und startet frpc.service.
  • Self-Update: per apt/dnf aus dem Aptly-Repo; Konfig und Service werden bei Upgrades nicht überschrieben.

Repository einrichten (Debian/Ubuntu)

Die DEB-Pakete liegen im internen Aptly-Repo, das der Installer beim ersten Start konfiguriert. Für manuelle Einrichtung:

bash
curl -fsSL https://apt.ks98.de/pubkey.asc | sudo gpg --dearmor \
    -o /etc/apt/keyrings/adminhelper.gpg

echo "deb [signed-by=/etc/apt/keyrings/adminhelper.gpg] \
https://apt.ks98.de/ stable main" | sudo tee /etc/apt/sources.list.d/adminhelper.list

sudo apt update
sudo apt install adminhelper-agent

Repository einrichten (Rocky/Alma/RHEL)

bash
cat <<'EOF' | sudo tee /etc/yum.repos.d/adminhelper.repo
[adminhelper]
name=AdminHelper
baseurl=https://rpm.ks98.de/stable
enabled=1
gpgcheck=1
gpgkey=https://rpm.ks98.de/pubkey.asc
EOF

sudo dnf install adminhelper-agent

Erst-Provisionierung

Nach Installation läuft der Agent noch nicht produktiv – er hat weder einen Server-URL noch ein FRP-Client-Zertifikat. Provisionierung in zwei Schritten:

  1. Im Web-Panel → Server → Neuen Server registrieren klicken. Das System legt einen Server-Eintrag an und generiert einen einmaligen Provision-Token (24h gültig).
  2. Auf dem Ziel-Server:
    bash
    sudo adminhelper-agent provision \
    --server https://srm.example.com \
    --token  <provision-token>

Der Agent ruft /api/frp/provision auf, bekommt Client-Zertifikat + frpc.toml, aktiviert adminhelper-agent.timer und frpc.service. Der Token wird nach Einlösung ungültig.

Ansible-One-Liner

Für Rollouts auf viele Hosts: ein Playbook mit apt/yum-Modul plus command: adminhelper-agent provision .... Beispiel-Role im Repo unter agent-go/ansible/.

Windows (MSI)

Die Windows-Binary adminhelper-agent.exe wird als signiertes MSI verteilt. Nach Installation:

powershell
adminhelper-agent.exe provision `
    --server https://srm.example.com `
    --token  <provision-token>

Der Agent registriert sich als Windows-Service (adminhelper-agent) und startet automatisch beim Boot.

Prüfen, ob der Agent läuft

bash
systemctl status adminhelper-agent.timer   # Linux – nächste Aktivierung
systemctl status frpc.service              # FRP-Client-Verbindung
journalctl -u adminhelper-agent.service --since "1 hour ago"

Upgrade

Bei gesetztem Apt-Repo reicht apt upgrade adminhelper-agent auf allen Servern. Die systemd-Units und die Konfig (/etc/adminhelper/agent.toml) bleiben erhalten.
