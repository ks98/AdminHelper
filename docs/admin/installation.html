Installation

SRM wird als Docker-Compose-Stack betrieben: Server, Redis, FRP-Server (frps), Monitoring-Service und VictoriaMetrics. Diese Seite führt durch eine frische Installation inklusive Reverse-Proxy-Setup und Erstanmeldung.

Voraussetzungen

  • Linux-Host mit Docker 24+ und Docker-Compose v2
  • Öffentlich erreichbarer Port 443 (bzw. 7000 und 7443 für FRP)
  • DNS-Eintrag, der auf den Host zeigt (srm.example.com)
  • Reverse-Proxy (nginx, Traefik, Caddy) für TLS-Termination – empfohlen

Stack starten

Die offizielle docker-compose.yml liegt im Repository-Root. Für einen schnellen Test reicht:

bash
git clone https://git.ks98.de/kevin/simpleremotemanager.git
cd simpleremotemanager

# .env anlegen
cat > .env <<'EOF'
DOMAIN=srm.example.com
SECRET_KEY=$(openssl rand -base64 48)
ADMIN_PASSWORD=$(openssl rand -base64 24)
TZ=Europe/Berlin
MONITOR_API_KEY=$(openssl rand -hex 32)
TRUSTED_PROXIES=172.17.0.1,10.0.0.0/8
EOF

docker compose pull
docker compose up -d
SECRET_KEY unbedingt setzen

Wenn SECRET_KEY leer ist oder den Default-Wert change-me-in-production hat, generiert der Server beim ersten Start einen zufälligen Key und speichert ihn in ./data/.secret_key. Das funktioniert, ist aber nicht portabel zwischen Hosts. Setze den Key immer explizit, wenn du den Stack migrieren oder replizieren willst.

Wichtige Umgebungsvariablen

VariableDefaultBeschreibung

Reverse-Proxy (empfohlen)

Der Server selbst bindet auf 8443 und präsentiert ein Self-Signed-Zertifikat. Für Produktion wird er üblicherweise hinter einem Reverse-Proxy mit Let's-Encrypt-Zertifikat betrieben. Beispiel für nginx:

nginx
server {
    listen 443 ssl http2;
    server_name srm.example.com;

    ssl_certificate     /etc/letsencrypt/live/srm.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/srm.example.com/privkey.pem;

    location / {
        proxy_pass         https://127.0.0.1:8443;
        proxy_ssl_verify   off;
        proxy_set_header   Host $host;
        proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;
    }
}

Dann in der .env:

.env
TRUSTED_PROXIES=127.0.0.1,::1

Erstanmeldung

  1. Web-Panel öffnen: https://srm.example.com
  2. Login mit admin + dem Wert aus ADMIN_PASSWORD
  3. Unter Benutzer einen eigenen Admin-Account anlegen, ausloggen
  4. Mit dem neuen Account anmelden und den initialen admin-Account deaktivieren
Health-Check

Der Server liefert unter /api/health einen kurzen JSON-Status. Nutze den Endpunkt für Monitoring-Systeme oder Docker-Healthchecks.

Updates

Der Server läuft aus dem Container-Registry docker.nevondo.com. Standard-Tags:

  • :latest – letzter erfolgreicher Build der Default-Branch
  • :dev – derselbe Build, explizit für Dev-Umgebungen getaggt
  • :v0.20.0 – Release-Tag (protected)
  • :<commit-sha> – deterministisches Image pro Commit
bash
docker compose pull server
docker compose up -d server

SQLite-Datenbank-Migrationen laufen automatisch beim Start (Schema-Versionen in server/app/db/migrations/). Der Agent hat einen eigenen Release-Zyklus – siehe Agent-Deployment.
