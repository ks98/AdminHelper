Admin-Handbuch

Übersicht

Dieses Buch richtet sich an Administratoren, die AdminHelper installieren, betreiben und im Team verteilen. Es erklärt Deployment, Benutzerverwaltung, Agent-Rollout, Monitoring und Troubleshooting – ausschließlich aus Betriebssicht. Code- und Architekturdetails findest du im Entwickler-Handbuch.

Komponenten auf einen Blick

Server

Docker-Container mit FastAPI + PostgreSQL. Liefert Web-Panel, REST-API, Monitoring-Scheduler und FRP-Tunnel-CA. Läuft typischerweise hinter einem Reverse-Proxy mit TLS.

Desktop-Client

Tauri-App für Linux, Windows und macOS. Holt Verbindungen vom Server und öffnet RDP-, SSH-, VNC- oder HTTP-Sessions inklusive FRP-Tunnel.

Go-Agent

Einheitliche Binary für Linux (systemd) und Windows. Übernimmt Monitoring-Collector-Läufe und FRP-Client-Auto-Provisioning. Bereitgestellt als DEB, RPM oder MSI.

Typische Admin-Aufgaben

  • Server per Docker-Compose installieren und erstmalig initialisieren
  • Admin-Benutzer erstellen, API-Keys verteilen
  • Agent auf neuen Servern via apt/dnf oder MSI ausrollen
  • Monitoring-Checks definieren und Alerts prüfen
  • FRP-Tunnel für einen Server einrichten
  • Probleme diagnostizieren und Logs lesen

Architektur in einem Absatz

Modularer Monolith

AdminHelper ist ein modularer Monolith: ein FastAPI-Prozess bedient das Web-Panel, die REST-API und den APScheduler, der Monitoring-Jobs ausführt. Metriken werden an ein externes VictoriaMetrics gestreamt, FRP läuft in einem eigenen Container. Persistenz: PostgreSQL 17 (eigener Compose-Service); Server- und Monitoring-Service teilen sich einen Postgres-Cluster mit zwei DBs (adminhelper, adminhelper_monitor).
