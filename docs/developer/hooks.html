Hooks & Events

Das Hook-System ist der Haupt-Integrationspunkt nach außen. Jedes relevante Ereignis im Server feuert ein Event, registrierte Scripts laufen in einer sanitisierten Subprozess-Umgebung mit harter Timeout-Grenze.

Architektur

tree
server/app/modules/hooks/
├── router.py          /api/hooks CRUD
├── models.py          Hook, HookExecution
├── schemas.py         Pydantic-Schemas
├── scheduler.py       Event-Dispatch + Retry-Queue
├── script_runner.py   Subprocess-Spawn (async)
└── script_worker.py   Child-Prozess: stdin=JSON, stdout/stderr capture

Event-Katalog

EventPayload (Auszug)
server.provisionedserver_id, hostname, ip
server.online / server.offlineserver_id, last_seen
connection.created / updated / deletedConnection-Objekt
connection.launchedconnection_id, user, client (desktop / web)
alert.triggered / resolved / acknowledgedcheck_id, severity, value
user.login / login_faileduser_id, ip, reason?
hook.failedMeta-Event bei Hook-Fehler (kann NICHT weitere Hooks triggern — Loop-Guard)

Hook-Definition

json
{
  "name": "Slack-Alert",
  "event": "alert.triggered",
  "script": "#!/usr/bin/env bash\nset -euo pipefail\nEVENT=$(cat)\ncurl -sS -X POST $SLACK_URL -d \"payload={\\\"text\\\": \\\"Alert: $(echo $EVENT | jq .check_name)\\\"}\"",
  "timeout_sec": 10,
  "enabled": true
}

Das Event-JSON wird per stdin übergeben – nicht per Argument oder Env-Var. Das vermeidet Shell-Escape-Probleme und ARG_MAX-Limits.

Sandbox-Modell

Hook-Scripts laufen in einem Fork mit geschärften Defaults:

  • Minimal-Env: Nur PATH, HOME, LANG, sowie explizit konfigurierte Secrets (SLACK_URL, etc.). Keine Weitergabe von SECRET_KEY, ADMIN_PASSWORD usw.
  • CWD ist ein tempfile.mkdtemp()-Verzeichnis, das nach Ablauf gelöscht wird.
  • Timeout über asyncio.wait_for + process.kill(), Default 10s, Max 300s.
  • Output-Limit: je 64 KiB für stdout und stderr; Rest wird abgeschnitten und im Log markiert ([truncated]).
  • User-Switch: Läuft als nobody, wenn Server mit Root-Caps deployed ist (Docker: --user 65534).
Kein vollständiger Sandbox

Die Hook-Isolation ist Defense-in-Depth, nicht ein Seccomp/Namespace-Jail. Admins mit Hook-CRUD-Rechten können beliebigen Code auf dem Server-Host ausführen. Hook-Zugriff daher nur vertrauenswürdigen Rollen geben.

Retry-Strategie

Schlägt ein Hook fehl (exit_code != 0 oder Timeout), wird er mit Exponential-Backoff erneut versucht: 30 s, 2 min, 10 min, 1 h. Nach dem vierten Fehlschlag wird HookExecution.state = failed gesetzt und das Meta-Event hook.failed gefeuert (Loop-Guard schützt vor Kaskaden).

Script-Runner-Implementierung

python
async def run_hook(hook: Hook, event: dict) -> HookExecution:
    payload = json.dumps(event).encode()
    env = _sanitised_env(hook)  # nur PATH, HOME, LANG, + explizite Secrets

    with tempfile.TemporaryDirectory() as cwd:
        proc = await asyncio.create_subprocess_exec(
            "/bin/sh", "-c", hook.script,
            stdin=asyncio.subprocess.PIPE,
            stdout=asyncio.subprocess.PIPE,
            stderr=asyncio.subprocess.PIPE,
            cwd=cwd, env=env,
        )
        try:
            out, err = await asyncio.wait_for(
                proc.communicate(payload),
                timeout=min(hook.timeout_sec, MAX_TIMEOUT),
            )
        except asyncio.TimeoutError:
            proc.kill()
            return HookExecution(state="timeout", ...)
        return HookExecution(state="ok" if proc.returncode == 0 else "failed",
                             stdout=out[:65_536], stderr=err[:65_536],
                             exit_code=proc.returncode)

Eigene Events feuern

Interner Code kann jederzeit Events feuern:

python
from app.modules.hooks import dispatch_event

await dispatch_event("connection.launched", {
    "connection_id": conn.id,
    "user": current_user.username,
    "client": "desktop",
})

dispatch_event ist non-blocking: Der Hook-Scheduler queued die Ausführung und kehrt sofort zurück. Die HTTP-Response des Auslösers wartet also nicht auf den Hook.

Test-Execution

Im Web-Panel kann jeder Hook mit einem Beispiel-Event geprüft werden: POST /api/hooks/{id}/test. Der Execution-Log zeigt stdout/stderr/exit_code in Echtzeit – praktisch beim Script-Debugging.
