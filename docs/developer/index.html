Entwickler-Handbuch

Übersicht

Das Entwickler-Handbuch dokumentiert Architektur, Code-Struktur und API von AdminHelper. Es richtet sich an alle, die am Code mitschreiben oder AdminHelper in ihr eigenes System integrieren wollen.

Tech-Stack auf einen Blick

Backend

Python 3.12, FastAPI, SQLAlchemy 2.0 + Alembic, APScheduler, PostgreSQL 17 via psycopg3. Strenge Pydantic-Schemas.

Web-Panel

Svelte 5 Runes, TypeScript strict, Vite, Hash-Router, uPlot, Playwright-E2E.

Desktop

Tauri 2 + Rust, Svelte 5 Frontend, keyring-Secrets, RDP-Race-Guard mit UUID.

Agent

Go 1.24, Cobra CLI, sqlx, native systemd + windowssvc-Integration.

Lokale Entwicklung

Jedes Unterprojekt ist eigenständig, kein Monorepo-Toolchain:

bash
# Server
cd server && uv sync && uv run uvicorn app.main:app --reload --port 8443

# Web-Panel (mit Proxy zum laufenden Server)
cd frontend-src && npm ci && npm run dev

# Desktop-Client (startet Tauri-Dev mit Auto-Reload)
cd desktop-src && npm ci
cd ../desktop && cargo tauri dev

# Go-Agent
cd agent-go && go run ./cmd/adminhelper-agent version
